Количество пакетов в нашем проекте растет, а мы тем временем все дальше уходим от учебных кусочков скриптов в сторону
настоящего тестового проекта, поэтому в этом шаге давайте зафиксируем все пакеты, которые мы используем. Это стандартная
практика, которая позволяет быстро переключаться в свежее виртуальное окружение, а также работать нескольким людям над
одним проектом, получая одинаковые результаты.

Откройте терминал, перейдите в директорию, в которой вы работаете с автотестами, и активируйте виртуальное окружение.

После чего выполните в терминале команду:

pip freeze > requirements.txt
Эта команда сохранит все версии пакетов в специальный файл requirements.txt.

Как их оттуда достать? Попробуйте создать новое виртуальное окружение (если нужно, вернитесь в модуль 1 за
инструкциями) и активировать. После чего выполните команду:

pip install -r requirements.txt
В свежем окружении все пакеты установлены одной командой!

Виртуальное окружение:
 Запустить (создать папку) виртуального окружения:
  python -m venv venv

 Активировать окружение:
  $ venv\Scripts\activate

 Печать установленных библиотек:
  pip freeze

 Обновить установленную библиотеку:
  pip install -U pip название_библиотеки

 Устанавливаю нужные библиотеки:
  pip install название_библиотеки еще_одной_библиотеки

 Удаление библиотеки:
  pip uninstall название_библиотеки -y

 Создаю файл с зависимостями:
  pip freeze > requirements.txt

 Отключить виртуальное окружение:
  deactivate

 Активировать и запустить новое окружение с новыми зависимостями:
  python -m venv venv
  $ venv\Scripts\activate
  pip install -r requirements.txt
